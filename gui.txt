Сущности:
- блок интерфейса
 - процедура. имеет параметры (входные, выходные) и опционально - поля, когда возвращает набор данных
  - поля считаются выходными параметрами, по текущей записи
  - входной параметр может быть ссылкой на форму, для выбора знаечния из списка
  - есть ссылки на другие блоки, открывающиеся в новом окне, в виде либо кнопок (пунктов меню) либо ссылок из грида
  - визуально имеет две части - показ входных параметров и грид с набором данных
   - эти две панели могут быть произвольно расположены относительно друг друга
   - входные параметры могут располагаться вертикально или горизонтально
   - набор данных может показываться как грид (строки) и как запись (все поля, с навигатором)
 - форма, контейнер блоков
  - входные, выходные параметры задаются вручную
  - входные параметры переносятся в заданные входные параметры блоков
  - выходные переносятся из выходных параметров выбранных блоков
  - может дополнительно настраиваться в коде (наследнике стандартной формы)
  - настройка визуализации блока настраивается только в контексте его присутствия в форме
  - в разных формах блок может иметь разный вид
  - для целей отображения блоки организованы в иерархию панелей
- меню приложения
 - это папки
 - и ссылки на формы

как пример - форма редактирования. это два блока:
- блок чтения записи. входной параметр - первичный ключ, выходные - поля объекта. блок спрятан (ui$form_child.visible=0)
- блок процедуры редактирования. входные параметры - поля объекта (биндятся из блока чтения, в таблице ui$form_child_param), выходных нет. в блоке показаны только входные параметры (т.к. ui$procedure.is_dataset=0), в вертикальном режиме
- при открытии формы все блоки датасетов запускаются в порядке, определяемом зависимостями между блоками (ui$form_child_param.source_block)
- при сохранении формы все блоки процедур выполняются со своими параметрами
- процедуры для автоматического создания процедур и форм редактирования и просмотра датасета по таблице с комментариями
